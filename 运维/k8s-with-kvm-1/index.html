why
#

本人宿主机环境是fedora rawhide。配有一个小的minikube集群用于微服务开发学习。但总是感觉跟实际环境有很大不同，

目标
#

个人机器，日常使用为了节能、速度，会避免配置成各种服务自启动的样子。

本机搭建一个用于分布式项目的三节点KVM集群。 目标： 1.三个节点在同一网段内透明通讯，并可以连接外部网。 2.易用的单个启动脚本。

坑点速览
#

virt-managaer非常推荐。本人过于菜鸡，GPT 折腾了半天想做脚本化启动三个 debian13.qrow 失败，明明用devstack配置好好的（但是不想上这么重），还是喜欢iso+图形化配置（虚拟机内仍是TTY only），最后回到舒适区选择了 rocky linux。总体结果是作为日常学习折腾的 k8s 环境还不错。

![[Pasted image 20260504195909.png]] rocky 的网络配置之类的相当现代化，软件包源，基本没什么坑点。

![[Pasted image 20260504195824.png]]

安装过程
#

安装虚拟化软件
# 

sudo dnf install -y qemu-kvm libvirt virt-install bridge-utils virt-manager

下载 Rocky10 minimal 安装
#

这部分使用virt-managaer图形化操作，minimal主要是方便以后配置。 只需要安装好一台。剩下的待会配置完克隆。

# 关闭防火墙（或者开放特定端口）
systemctl disable --now firewalld

# 关闭 SELinux
setenforce 0
sed -i 's/^SELINUX=enforcing$/SELINUX=permissive/' /etc/selinux/config

# 关闭 Swap（临时+永久）
swapoff -a
sed -i '/swap/d' /etc/fstab

宿主机防火墙放行
#
